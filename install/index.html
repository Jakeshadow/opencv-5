How to Install OpenCV 5

OpenCV 5 released June 2026. pip install opencv-python>=5.0.0 is all you need for Python. C++ requires C++17 and CMake 3.20+. Full guide for every platform.

Quick Start (pip)

pip install opencv-python>=5.0.0

# Verify
python -c "import cv2; print(cv2.__version__)"
# Expected: 5.0.0

# If you need contrib modules (cv2.ml, Haar, SURF, G-API)
pip install opencv-contrib-python>=5.0.0

Platform-Specific Guides

PlatformMethodNotes
Windows pip install opencv-python>=5.0.0 If you had opencv-python-headless before, uninstall all opencv packages first — mixing flavors causes DLL conflicts. See fix below.
macOS pip install opencv-python>=5.0.0 Works on both Intel and Apple Silicon. For M1/M2/M3, the wheel includes universal2 binaries.
Linux (Ubuntu 22.04+) pip install opencv-python>=5.0.0 Ubuntu 20.04 / CentOS 7 / Debian 10 users: pip wheel needs GLIBCXX_3.4.29 — see GLIBCXX fix below.
Jetson / ARM Source build No pre-built pip wheel for ARM64 yet. Build from source with C++17 toolchain.
Docker FROM python:3.12-slim Use ubuntu:22.04+ as base image. Dockerfile example below.

Source Build (CMake)

# Clone
git clone -b 5.x https://github.com/opencv/opencv.git
cd opencv && mkdir build && cd build

# Configure — must use C++17
cmake -DCMAKE_CXX_STANDARD=17 \
      -DCMAKE_BUILD_TYPE=Release \
      -DBUILD_EXAMPLES=ON \
      ..

# Build (adjust -j to your core count)
make -j$(nproc)

# Install
sudo make install

# Verify
pkg-config --modversion opencv5
# Expected: 5.0.0

Key CMake flags:

  • -DBUILD_opencv_contrib=ON — include contrib modules
  • -DWITH_CUDA=ON — CUDA support (classic engine only in v5)
  • -DCMAKE_INSTALL_PREFIX=/usr/local/opencv5 — install side-by-side with v4

Docker Setup

# Dockerfile
FROM python:3.12-slim

RUN pip install opencv-python>=5.0.0

# Verify
RUN python -c "import cv2; print(cv2.__version__)"

# Your app
COPY app.py .
CMD ["python", "app.py"]

For GPU Docker images, use nvidia/cuda:12.4-runtime-ubuntu22.04 as base instead. Remember the new DNN engine is CPU-only — pin ENGINE_CLASSIC for GPU.

conda Install

# conda-forge (may lag behind pip by a few days)
conda install -c conda-forge opencv=5.0.0

# If not available yet, use pip inside conda env
conda create -n cv5 python=3.12
conda activate cv5
pip install opencv-python>=5.0.0

Common Install Errors & Fixes

Windows: "ImportError: DLL load failed"

You had opencv-python-headless installed and installed opencv-python on top (or vice versa). The two packages ship different DLL sets.

pip uninstall opencv-python opencv-python-headless opencv-contrib-python -y
pip install opencv-python>=5.0.0
# Pick exactly ONE flavor — never mix headless with GUI version

Linux: "GLIBCXX_3.4.29 not found"

OpenCV 5 wheels are compiled with C++17 and need a newer libstdc++ than what ships on Ubuntu 20.04 / CentOS 7 / Debian 10.

# Ubuntu 20.04
sudo apt install gcc-12 g++-12

# conda (bundles compatible libstdc++)
conda install -c conda-forge libstdcxx-ng

# Docker — switch base image
# Bad:  FROM python:3.12-slim (based on Debian 10)
# Good: FROM ubuntu:22.04 + install python3

macOS: "Library not loaded: @rpath/libopencv..."

Typically a Homebrew conflict. The system finds Homebrew's OpenCV 4 dylib before the pip-installed v5.

# Uninstall Homebrew's OpenCV if present
brew uninstall opencv

# Then reinstall from pip
pip install --force-reinstall opencv-python>=5.0.0

ModuleNotFoundError: No module named 'cv2.ml'

cv2.ml moved to opencv_contrib in v5. Install opencv-contrib-python instead of opencv-python, or switch to scikit-learn.

pip uninstall opencv-python -y
pip install opencv-contrib-python>=5.0.0
