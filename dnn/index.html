OpenCV 5 DNN: Hands-On with the New Engine

The v5 DNN module gets a ground-up rewrite — 80%+ ONNX operator coverage, graph-based engine with attention fusion, native LLM/VLM inference. Three complete examples with copy-paste-ready code.

What Changed in the DNN Module

The Short Version

cv2.dnn.readNet("model.onnx") still works. But underneath is a completely new graph-based engine that handles dynamic shapes, fuses attention patterns, and runs models that used to fail with "unsupported operator." It's CPU-only for now — GPU users stay on ENGINE_CLASSIC.

Example 1: YOLOv8 Object Detection

import cv2
import numpy as np

# Load model — ENGINE_AUTO picks new engine, falls back to classic
net = cv2.dnn.readNet("yolov8n.onnx")

# Read and preprocess
img = cv2.imread("street.jpg")
blob = cv2.dnn.blobFromImage(img, 1/255.0, (640, 640), swapRB=True)

net.setInput(blob)
output = net.forward()

# Parse detections (simplified — full parsing logic in YOLO docs)
# output shape: [1, 84, 8400] for YOLOv8n
# 84 = 4 bbox coords + 80 class scores
boxes = output[0, :4, :].T  # [8400, 4]
scores = output[0, 4:, :].max(axis=0)  # [8400]
class_ids = output[0, 4:, :].argmax(axis=0)  # [8400]

# Filter by confidence
mask = scores > 0.5
boxes = boxes[mask]
scores = scores[mask]
class_ids = class_ids[mask]

print(f"Detected {len(boxes)} objects")
for i, (box, score, cls) in enumerate(zip(boxes, scores, class_ids)):
    print(f"  [{i}] class={cls} score={score:.2f} box={box}")

This code works identically in OpenCV 4 and 5. The difference: v5's new engine handles YOLOv8's dynamic output shapes natively — no manual reshape hack needed on some export variants.

Example 2: ResNet Image Classification

import cv2
import numpy as np

net = cv2.dnn.readNet("resnet50.onnx")

img = cv2.imread("cat.jpg")
blob = cv2.dnn.blobFromImage(img, 1/255.0, (224, 224),
                              mean=(0.485, 0.456, 0.406),  # ImageNet stats
                              swapRB=True)

net.setInput(blob)
probs = net.forward()  # shape: [1, 1000]

top5 = np.argsort(probs[0])[::-1][:5]
for i, idx in enumerate(top5):
    print(f"  Top {i+1}: class={idx} prob={probs[0][idx]:.4f}")

Example 3: Vision Transformer (ViT) Inference

import cv2
import numpy as np

net = cv2.dnn.readNet("vit_base_patch16_224.onnx")

img = cv2.imread("dog.jpg")
blob = cv2.dnn.blobFromImage(img, 1/255.0, (224, 224),
                              mean=(0.5, 0.5, 0.5),
                              swapRB=True)

net.setInput(blob)
output = net.forward()  # shape: [1, 1000]

pred_class = np.argmax(output)
print(f"Predicted class: {pred_class}, confidence: {output[0][pred_class]:.4f}")

ViT models often failed in OpenCV 4 due to unsupported attention operators. In v5, the new engine's attention fusion handles them — no more "Unsupported op type: Attention" errors.

Fallback Strategy

When the New Engine Fails

ENGINE_AUTO (default) tries the new engine first, falls back to classic silently. But some models compile without error yet produce wrong output — the "silent killer" scenario. 

# Explicitly pin the classic engine for GPU inference
net = cv2.dnn.readNet("model.onnx")
net.setPreferableBackend(cv2.dnn.DNN_BACKEND_INFERENCE_ENGINE)
net.setPreferableTarget(cv2.dnn.DNN_TARGET_CPU)

# Or for CUDA (classic engine only in v5)
net.setPreferableBackend(cv2.dnn.DNN_BACKEND_CUDA)
