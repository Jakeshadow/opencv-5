OpenCV 5 vs 4 Performance Benchmarks: Real Speed Numbers

How much faster is OpenCV 5 really? DNN inference, image processing, and ONNX models — benchmarked on real hardware.

DNN Inference: The Biggest Win

The new graph-based DNN engine is the standout improvement. ONNX model inference on CPU is 27–37% faster across the board.

ModelOpenCV 4.10 (ms)OpenCV 5.0 (ms)Speedup
YOLOv8-n (640)42126537% faster
YOLOv8-m seg (640)25518527% faster
MobileNetV2 (224)~2.01.3533% faster
ResNet-50 (224)281932% faster
EfficientNet-B0 (224)1510.530% faster

Tested on Intel i7-13700H, 32 GB RAM, Windows 11, ONNX models, ENGINE_AUTO. Each result is the median of 100 inferences after 5 warmup runs.

Key takeaway: The new engine matches or beats ONNX Runtime on CPU for most CNN models. The 80% ONNX operator coverage (vs. 22% in v4) means far fewer models need the classic engine fallback.

When to Stay on OpenCV 4

OpenCV 5 is faster nearly everywhere, but stay on v4 if you have any of these:

  • GPU-accelerated DNN with CUDA/OpenCL (v5 new engine is CPU-only for now)
  • Production code relying on Caffe or Darknet model parsers
  • Pixel-exact test baselines (v5 interpolation and text rendering differ)
  • Legacy C++ code using the old C API (CvMat/IplImage)
